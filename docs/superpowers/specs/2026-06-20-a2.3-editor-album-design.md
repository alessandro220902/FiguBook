## Modello dati

### Sezioni (porto da `album-data-fwc2022.js`)
Struttura per sezione: `{ id, name, short, group, kind, codes[], c1, c2 }`.
- `group` = raggruppamento (es. "Girone A") per la sidebar.
- `kind` = `team` | `opening-trofei` | `other`.
- `c1`/`c2` = due colori squadra → gradiente identità + **motore colore dinamico**.
- `codes[]` = codici figurina (es. `QAT1`…`QAT20`).
- `STICKER_NAMES` (parziale) e `STICKER_STATES` (legacy) — i nomi si portano; lo stato iniziale arriva da Firestore, non dal file.

Si riscrive in modulo TS **senza `window` globali**: es. `src/data/albums/fwc2022.ts` che esporta `sections`, `groups`, `names`. Tipi in `src/data/albums/types.ts`.

### Stato figurina = contatore
**`count: number`** per ogni codice (0 = mai posseduta). Stati **derivati**, non memorizzati:
- `count === 0` → **mancante**
- `count >= 1` → **posseduta**
- `doppie = max(0, count - 1)` → es. `count 3` = posseduta + `×2` doppie

Le statistiche (sezione e album) si calcolano dai count:
- Possedute = `#{codes: count>=1}`
- Mancanti = `#{codes: count===0}`
- Doppie (totale copie extra) = `Σ max(0, count-1)`
- Completamento % = `possedute / totale`

Firestore: `users/{uid}/albums/{albumId}` = `{ counts: Record<code, number>, updatedAt }`. (Niente map `states` separata: ridondante e fonte di drift — la si deriva.)

## Struttura pagina `/album/:albumId`

### Piano 1 — Landing hero
- **Copertina album** + **completamento %** (badge sulla copertina). **Copyright-safe**: la copertina è arte propria/generica, mai scansioni di album o figurine Panini (immagini protette + loghi/likeness).
- Stat a destra: **Possedute X / totale · Mancanti · Doppie**.
- Bottoni: **Condividi doppie**, **Condividi mancanti** → STUB disabilitati (tooltip "presto").
- Ingresso d'effetto: `container-scroll-animation` (21st/aceternity) — la copertina si raddrizza/scala allo scroll. Con `prefers-reduced-motion` → reveal statico. Perf: la pagina si visita spesso, l'effetto non deve bloccare l'editor sotto.
- Scroll/CTA → editor a sezioni.

### Piano 2 — Editor a sezioni
- **Sidebar Sezioni**: search (sezione o #figurina), lista per `group` → sezioni, ogni voce con mini-crest gradiente, nome, `possedute/totale` + %. Voce attiva ritinta dai colori sezione.
- **Hero sezione**: kicker `Sezione NN · Girone X`, nome, range codici + conteggio, pill `Possedute`/`Mancanti`, completamento % + barra. Sfondo = gradiente `c1→c2` della sezione.
- **Toolbar**: filtri `Tutte / Mancanti / Doppie / Possedute` (con contatori) · toggle **Inserimento rapido** · segmented **Griglia / Lista**.
- **Griglia**: card per figurina (vedi sotto).

### StickerCard (card figurina)
- **Mancante** (`count 0`): card spenta/tratteggiata, `+` (appare su hover/focus).
- **Posseduta** (`count >=1`): **fill pieno gradiente squadra**, codice grande, badge `1` (o `count`?) in angolo, nome su hover (se disponibile), bottone **meno** per `−1`.
- **Doppie** (`count >=2`): come posseduta + badge `×(count-1)` ambra.
- **Stato leggibile sempre**: distinto per **luminosità + icona/badge**, mai dal colore squadra (daltonici). Card colore = identità; stato = luminosità.

## Interazione

| Contesto | Azione | Effetto |
|---|---|---|
| Inserimento **ON** | tap card | `count += 1` |
| Inserimento ON/OFF | bottone **meno** | `count -= 1` (min 0) |
| Inserimento **OFF** | tap card | apre **overlay info figurina** |
| Overlay info | bottone **Aggiungi** / **meno** | `count ±1` |

Overlay info: codice, nome (o codice), count attuale, squadra/sezione, copie/doppie. Niente dati sensibili. Chiusura via backdrop/Esc.

## Motore colore dinamico
Una sezione attiva imposta `--t1`/`--t2` (CSS custom properties) a livello del contenitore editor; hero, voce sidebar attiva, card possedute, barra progresso, filtro attivo derivano da lì (`color-mix`). Cambio sezione = ritintura (crossfade accento). Sorgente: `c1`/`c2` reali della sezione.

## Store ottimistico + batch (fix B1)
Problema vecchio (`figubook-db.js` ~189 `saveCardState`): **una scrittura Firestore per ogni tap**. Con conteggi rapidi = costoso e lento.

Design:
- `useAlbum(albumId)` tiene `counts` in stato locale + coda dei delta non ancora persistiti.
- Tap/meno → **update ottimistico immediato** della UI.
- Persistenza con **debounce** (~800ms di quiete) **o** flush su: cambio sezione, navigazione via, `visibilitychange`/`beforeunload`. Flush = `writeBatch` con tutti i delta accumulati (merge sul doc album).
- **Rollback** mirato su errore di scrittura: ripristina i count falliti + toast errore (riusa il pattern `harden` già introdotto in dashboard/useCollection).

## Error handling & edge cases
- **Caricamento dati album fallito** → stato errore + retry (no falso "vuoto"), pattern già usato.
- **Scrittura batch fallita** → rollback dei delta + toast, la UI resta coerente.
- **count negativo** → clamp a 0 (il meso non scende sotto).
- **Codici senza nome** → fallback al codice.
- **Sezione vuota / album sconosciuto** → empty state esplicito.
- **Offline** → ottimistico regge la UI; flush ritenta al ritorno online.
- **`prefers-reduced-motion`** → niente scroll-3D, niente shimmer.

- Debounce flush ~800ms + flush su navigazione/blur/visibilità.
